# 專案：FB_ScamHunter - 即時社群廣告防詐系統
## 核心價值 (Core Value)
本專案是在 AIS3 Hackathon 競賽中開發的即時防詐標記工具後端系統。
- 目標使用者：Facebook 使用者，特別是容易受到「贊助」廣告誤導的社群族群。
- 主要目的：建立高效能 API 服務，接收前端瀏覽器擴充元件傳回的廣告內容，並即時辨識是否為詐騙廣告。
- 功能願景：透過自動化的機器學習判定，在廣告出現的第一時間提供預警，防止使用者點擊惡意連結。

## 技術架構 (Technologies)
系統採用現代後端框架與高效能分類模型，確保低延遲的辨識體驗：
- 後端框架：使用 FastAPI 作為 API Gateway，專門處理高併發的辨識請求與回應。
- 機器學習模型：核心分類器為 XGBoost Classifier，用於對廣告內容進行惡意機率判定。
- 特徵工程與 NLP：結合 NLP 語意分析、內文結構特徵提取，以及圖像辨識輔助分析。
- 數據來源：
    - 詐騙數據：由 AIS3 Hackathon 提供，包含 5501 筆真實詐騙廣告資料。
    - 正常數據：從 Meta 廣告檔案庫 爬取之合法廣告數據。

## 開發動機 (Development Motivation)
- 解決社群詐騙氾濫：針對 Facebook 上日益增多的虛假贊助廣告，提供即時的技術反制手段。
- 整合自動辨識與視覺提醒：為了補足傳統黑名單更新過慢的問題，透過機器學習特徵來捕捉新型態詐騙。
- 學術與實戰結合：在 AIS3 競賽壓力下，設計一套具備擴充性且能處理真實場景數據的防禦系統。

## 遇到的挑戰與解決方案 (Challenges & Solutions)
### 挑戰 1：跨維度特徵的關聯分析
- 問題：單純的關鍵字過濾容易被詐騙者透過特殊字元規避。
- 解決方案：實作深度特徵工程，包含文字長度 (text_length)、數字出現頻率 (text_digit_count) 以及風險關鍵字統計 (risk_keyword_count)，從結構層面判定異常。
### 挑戰 2：廣告一致性驗證
- 問題：許多詐騙廣告會使用縮網址或偽造顯示網域來欺騙使用者。
- 解決方案：在後端加入廣告連結與顯示網域的一致性檢查邏輯，強化對釣魚連結的辨識能力。
### 挑戰 3：即時處理效能需求
- 問題：使用者滑動 FB 頁面時會產生大量請求，後端若處理過慢會影響使用者體驗。
- 解決方案：利用 FastAPI 的非同步特性處理請求，並將特徵工程與模型推理優化，確保在接收到 Request 後能快速回傳判定結果 (Verdict)。

## 專案亮點 (Project Highlights)
- 卓越的模型表現 (Model 5)：模型準確度 (Accuracy) 達 0.92，精確率 (Precision) 達 0.93。
- 完整的前後端運作流程：前端擷取 DOM 結構中的廣告內容。後端進行風險詞與特徵分析。XGBoost 給出結果，前端即時在 FB 頁面上標記紅框與提醒標籤。
- 支援手動查驗：提供 API 接口，允許使用者手動輸入可疑連結進行內文查驗。
- 具備未來擴充性：後端架構支援接入圖像 OCR 強化，或進一步同步 GASA 與 Whoscall 等外部防詐資料庫。
