# 專案：SelfMap - 打造你專屬的旅行地圖
## 核心價值 (Core Value)
SelfMap 是一個以個人化為核心的地理資訊標記平台 。
- 目標使用者：希望記錄私房景點、撰寫深度遊記，並與旅伴協作專屬地圖的旅行者 。
- 主要目的：透過直觀的介面，將每一次旅行的足跡與想法轉化為珍貴的回憶地圖 。
- 功能願景：提供使用者一個整理個人旅遊發現的工具，並能與他人共享故事 。

## 技術架構 (Technologies)
本專案採用現代前端技術棧與雲端後端服務構建：
- 前端框架：使用 React 18 配合 React Router DOM 進行頁面導航管理。
- 地圖 API：外接 Leaflet 與 OpenStreetMap 實現動態地圖交互，包括放大縮小、座標定位與地標顯示。
- 使用者介面：採用 Material UI (MUI) 與 Styled Components 打造專業且支持深淺色主題切換的 UI。
- 後端服務：核心數據與驗證由 Firebase 支援，包含 Firebase Authentication 用於會員管理，以及 Firestore 作為即時同步的資料庫。
- 開發工具：利用 Vite 進行快速建置，並透過 GitHub Pages 部署靜態網頁。
- Vibe Coding：在開發過程中深度結合 ChatGPT 5.1 輔助程式碼編寫與邏輯優化 。

## 開發動機 (Development Motivation)
- 個人化記錄需求：希望能捕捉並保存個人獨特的旅行發現，而非僅依賴大眾化的地圖標籤 。
- 旅伴協作痛點：為了解決多人在規劃行程時資訊分散的問題，設計了可共享連結，方便旅伴共同編輯地圖 。
- 情感連結與昇華：透過撰寫評論與遊記功能，深化每段旅程的情感價值，讓回憶更具體化 。

## 遇到的挑戰與解決方案 (Challenges & Solutions)
### 挑戰 1：即時數據同步與通知系統
- 問題：如何在多人協作時，讓一方新增的地標立即出現在所有人的地圖上？
- 解決方案：利用 Firebase Firestore 的 onSnapshot 特性，實作即時資料監聽。當資料庫異動時，前端會立即更新地標顯示與最新消息列表。
### 挑戰 2：圖片儲存與載入效能優化
- 問題：地標照片上傳可能導致載入速度變慢或資料庫負擔過重。
- 解決方案：實作了圖片壓縮與 Base64 處理機制，在客戶端先行壓縮圖片後再存儲，優化數據傳輸效率與儲存空間。
### 挑戰 3：地圖操作的精準度與自動化
- 問題：如何在眾多地標中快速找到特定地點並進行導航？
- 解決方案：開發了「地標列表視窗」，點擊即可自動定位至該座標；同時整合 Google Maps 導航 功能，使用者點擊地標卡片即可直接開啟手機導航。

## 專案亮點 (Project Highlights)
- 多維度地標管理：支援定位拍照或手動上傳圖片，並可為每個地標加入標題、地址、備註與評論。
- 靈活的共享機制：一鍵產生地圖分享連結，讓朋友即便沒有帳號也能查看指定的旅行軌跡。
- 動態通知系統：針對使用者操作記錄事件（如新增地標），並在「最新消息」中即時提醒使用者與其關注者。
- 完整的安全性設計：除了基本的註冊登入，還具備密碼重設郵件寄送與帳號刪除等完整的帳號管理流程。
