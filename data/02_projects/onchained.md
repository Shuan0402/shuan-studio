# 專案：OnChainED - 區塊鏈學習認證與教育記錄平台
## 核心價值 (Core Value)
OnChainED 是一個基於區塊鏈技術的去中心化教育認證平台，旨在解決傳統學歷證書易遭竄改與難以國際驗證的問題 。
- 目標使用者：學生、教育機構（如 Plan B Summer School）以及企業招聘方 (HR)。
- 主要目的：透過 DID + VC 技術，解決傳統證書易造假、驗證成本高的痛點。
- 功能願景：實現「自主主權身份」(Self-Sovereign Identity)，讓學歷證明不依賴中心伺服器。

## 技術架構 (Technologies)
本專案整合了 Web3 前端技術與去中心化儲存協議，構建完整的信任鏈結：
- 前端框架：使用 React 構建核心應用程式（App.jsx） 。
- 身份管理：透過 MetaMask 或 WalletConnect 進行錢包登入，作為 DID (did:ethr) 的基礎識別 。
- 憑證標準：採用 W3C VC (Verifiable Credentials) 與 JSON-LD 格式，並在 Ethereum/Polygon 鏈上進行簽章發證 。
- 儲存協議：利用 IPFS (Pinata) 或 Arweave 進行憑證內容的永久存證，確保資料不因伺服器關閉而消失 。
- 驗證工具：整合 SpruceID 建立符合國際規範的去中心化身份憑證 。

## 開發流程 (Development Process)
專案涵蓋了從身份建立到第三方驗證的完整閉環 ：
1. DID 建立：使用者透過 Web3 錢包登入後，系統自動產生唯一的去中心化身份識別碼 。
2. VC 簽發：教育機構端（IssueVC.jsx）輸入學員資料後，利用機構錢包簽署 JSON-LD 憑證 。
3. 分發與儲存：將簽署後的憑證上傳至 IPFS，並可透過 NFT Token Gate 或鏈上訊息傳送至學生空間 。
4. 即時驗證：第三方機構（如 HR）透過 VerifyVCFromQR.jsx 掃描證書 QR Code，無需中介即可驗證其原始簽章與可信度 。

## 遇到的挑戰與解決方案 (Challenges & Solutions)
### 挑戰 1：憑證的永久可視性與防竄改
- 問題：傳統數位證書若存儲於中心化伺服器，一旦平台結束營運，證書將無法讀取。
- 解決方案：採用 IPFS / Arweave 永久存證方案，配合區塊鏈鏈上簽章，確保證書具備跨時間與跨平台的持久性與真實性 。
### 挑戰 2：身份識別的自主權
- 問題：如何確保學生能完全掌控自己的學習記錄，而不被教育機構單方面註銷或鎖定？
- 解決方案：導入 DID (Decentralized Identifiers) 技術，讓學生以錢包作為身份根基，自主儲存、展示與分享其 VC 憑證 。
### 挑戰 3：驗證流程的便捷度
- 問題：區塊鏈驗證流程對非技術背景的 HR 而言可能過於複雜。
- 解決方案：開發 QR Code 驗章工具 與簡易的 VerifyVC 介面，讓驗證方只需掃描即可即時取得 JWS 驗章結果，降低進入門檻 。

## 專案亮點 (Project Highlights)
- 國際標準兼容性：嚴格遵循 W3C VC 規範，並能與 MIT Media Lab 的 Blockcerts 等既有國際專案進行效能與架構比較 。
- 三方受益模式：學生獲得自主履歷、機構降低發證成本、雇主大幅縮短學歷查驗時間 。
- 法規與環境契合：針對瑞士與歐洲區域的法規友善環境設計，具備高度的落地可行性 。
- 未來 AI 整合力：架構預留了 API 接口，未來可搭配 AI 模型進行履歷自動評分與防詐欺行為分析 。
