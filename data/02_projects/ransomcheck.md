# 專案：基於 API 呼叫序列之深度學習勒索病毒偵測系統
## 核心價值 (Core Value)
本專案開發了一種基於深度學習的動態分析模型，專門用於精準識別 Windows 平台上的勒索病毒。
- 目標使用者：網路安全研究人員、企業系統管理員，以及需要評估可疑檔案安全性的開發者或一般使用者。 
- 主要目的：透過監控軟體執行時的 API 呼叫行為，克服傳統靜態特徵碼偵測對變種病毒與混淆技術偵測能力不足的問題。 
- 功能願景：提供一個直觀且自動化的網頁平台，將複雜的惡意軟體分析轉化為清晰的安全性建議。 

## 技術架構 (Technologies)
本專案結合了自動化沙盒、深度學習特徵提取與現代網頁部署技術：
- 動態分析與環境：利用 Cuckoo Sandbox 於 Ubuntu 環境中建立隔離的 Windows 7 虛擬機，捕捉 PE 檔案的執行行為。 
- 深度學習模型架構：
    - Transformer：核心 NLP 技術，利用自注意力機制 (Self-Attention) 分析 API 呼叫序列間的語義關聯與長期依賴。 
    - CNN (卷積神經網路)：用於捕捉 API 序列中的局部行為模式與局部特徵。 
    - 嵌入技術 (Embedding)：將數值化的指令轉換為高維向量，使模型能理解不同指令間的相關性。 
- 網頁平台開發：
    - 前端：使用 TypeScript 與 React 構建具備高度模組化與組件重用性的介面。 
    - 後端：採用 Python Flask 輕量化框架，負責串接沙盒分析、模型推理與數據傳輸。 
- 開發與運算工具：利用 NVIDIA GeForce RTX 3060 進行高效能模型訓練與參數調整。 
    
## 開發動機 (Development Motivation)
- 全球攻擊情勢嚴峻：據 FortiGuard 報告，勒索病毒攻擊比例在半年內增長 13 倍，且組織成功檢測率持續下降，顯示威脅已日益複雜。 
- 補足傳統偵測缺口：早期的靜態分析 (Signature-based) 難以應對透過加密技術、規避手段產生的變種病毒，需透過動態行為模式進行辨識。 
- 語義資訊的深度挖掘：旨在透過 API 的「執行順序」與「語義細節」（如對系統的影響程度），而非僅依賴呼叫頻率來區分良性與惡意行為。 

## 遇到的挑戰與解決方案 (Challenges & Solutions)
### 挑戰 1：原始 API 序列的冗餘與混淆
- 問題：攻擊者常會插入大量重複或無意義的 API 呼叫來增加數據噪點，這會干擾特徵提取。 
- 解決方案：實作了資料預處理機制，包含刪除連續重複項與序列長度標準化（透過 Padding 與 Truncating），有效減少冗餘並確保輸入規格一致。 
### 挑戰 2：指令 ID 化導致的語義缺失
- 問題：直接將 API 名稱轉為數值 ID (直接轉換) 雖快速，但模型無法從 ID 中得知該操作的具體語義。 
- 解決方案：引入 四元組語義表示法 ($S_i = <action_i, object_i, class_i, category_i>$)，將操作動作、對象、影響程度與功能分類量化，提升模型的深層理解力。 
### 挑戰 3：捕捉行為序列的長期關聯
- 問題：勒索病毒的行為往往橫跨較長的執行週期，傳統 RNN 模型可能面臨梯度消失或記憶不足。 
- 解決方案：在分類器前置入 Transformer 編碼器，經測試證明其穩定性與辨識效能優於 LSTM 與 GRU，能更有效地分析 API 呼叫間的依賴關係。 

## 專案亮點 (Project Highlights)
- 卓越的辨識效能：模型測試結果極其出色，達到 $Accuracy$ 0.9994、$F1 Score$ 0.9996，且召回率 (Recall) 達 1.0。
- 全自動化分析流程：使用者只需於網頁介面上傳 PE 檔案，後端自動完成「沙盒模擬 → 數據預處理 → 模型預測 → 結果可視化」。
- 多層次特徵處理：結合了兩層嵌入層與三層不同尺寸的卷積核，同時處理「直接轉換」與「四元組轉換」的數據，確保特徵捕捉無死角。 
- 高安全性的測試機制：所有樣本均在乾淨且獨立的虛擬環境中執行，分析完畢後自動回復快照，防止惡意程式擴散並維持環境純淨。
