# 專案：Elecookie - 多角色餅乾電商平台
## 核心價值 (Core Value)
Elecookie 是一個專為餅乾產業設計的全方位電商解決方案，強調嚴謹的業務邏輯與細緻的角色權限管理。
- 目標使用者：餅乾愛好者（消費者）、店鋪員工以及平台系統管理員。
- 主要目的：提供從商品瀏覽、購物車操作到後端訂單流轉與店鋪管理的完整電商閉環。
- 功能願景：透過多層次的身份驗證，確保不同角色的使用者都能在同一個平台上高效協作，並保證訂單處理的精準度。

## 技術架構 (Technologies)
本專案採用前後端分離架構，結合了強類型語言的穩定性與前端框架的靈活性：
- 後端開發：使用 ASP.NET Core 框架構建 RESTful API，負責核心業務邏輯處理。
- 前端框架：採用 React.js 打造響應式使用者介面，提供流暢的購物體驗。
- 資料庫系統：選用 PostgreSQL 進行數據持久化，管理複雜的會員、商品與訂單關聯。
- 開發規範：嚴格遵循 TDD (Test-Driven Development) 流程，透過大量單元測試確保代碼品質。

## 開發動機 (Development Motivation)
- 實踐軟體工程嚴謹性：核心目標不在於功能的繁瑣，而是在於透過 TDD 流程 實踐「先測試、再開發、後重構」的品質要求。
- 複雜權限管理需求：挑戰在同一系統中區分「一般使用者、會員、員工、管理員」四種角色，並賦予差異化的操作權限。
- 確保訂單穩定流轉：電商系統的痛點在於訂單狀態轉換的正確性，本專案旨在透過單元測試與持續重構，打造穩定且可追蹤的訂單系統。

## 遇到的挑戰與解決方案 (Challenges & Solutions)
### 挑戰 1：多重身份權限與路由保護
- 問題：系統需同時支援四種角色，如何確保普通用戶無法進入店鋪管理後台，且管理員具備最高權限？
- 解決方案：在 ASP.NET Core 中實作基於角色的存取控制 (RBAC)，並與 React 前端路由結合，根據身份動態渲染導覽列與功能按鈕，實現嚴格的權限隔離。
### 挑戰 2：複雜業務邏輯下的程式碼穩定性
- 問題：隨著功能增加（如購物車、追蹤商店、訂單狀態更新），邏輯分支變得複雜，容易產生 Regression Bug。
- 解決方案：落實 TDD 開發模式，在撰寫功能前先寫測試案例。這使得在進行「持續重構」優化效能時，能確保原有訂單流轉與核心邏輯不受到破壞。
### 挑戰 3：多元化的商品檢索需求
- 問題：使用者需要依照分類、店家、或詳細資訊等多種維度進行查詢，資料庫查詢效能與準確性變得至關重要。
- 解決方案：設計了多維度的查詢接口，支援「依照分類查詢所有商品」與「查看特定店家詳情及分類商品」，優化資料庫索引以提升查詢速度。

## 專案亮點 (Project Highlights)
- 全方位的會員與商店系統：具備完整的註冊/登入、修改資料、以及獨特的「追蹤商店」功能，增強使用者黏著度。
- 專業級店鋪管理工具：員工與管理員可進行店鋪資訊修改、商品 CRUD（增刪改查）與實時訂單檢視，具備完整的 B2C 營運能力。
- 高品質代碼保證：透過單元測試與重構，確保了系統在處理大量訂單流轉時的穩定度，展現對軟體穩定性的極高要求。
- 清晰的角色界定：精準切分「未登入使用者」與「正式會員」的功能差異，並提供員工專屬的工作台介面。
